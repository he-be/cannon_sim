# 今後の展望 (Roadmap)

## 短期的な改善 (Short-term Improvements)

### 1. コードの整理とリファクタリング

- **不要なコードの削除**: `LeadAngleCalculator` など、古い実装の名残で現在使われていないメソッドやクラスを整理する。
- **型定義の強化**: `any` 型の使用を排除し、より厳密な型チェックを行う。

### 2. テストの拡充

- **エッジケースのテスト**: 極端な角度、超遠距離、高速移動目標などに対するテストケースを追加。
- **パフォーマンス回帰テスト**: 最適化されたShooting Methodのパフォーマンスが維持されているか定期的にチェックする仕組み。

## 中長期的な目標 (Mid-to-Long-term Goals)

### 1. 物理モデルの高度化

- **コリオリ力**: 地球の自転による影響を考慮（長距離射撃で重要）。
- **風の影響**: 風向・風速をランダムまたは設定可能にし、シミュレーションに反映。
- **空気密度の高度変化**: 高度による空気密度の変化を実装（現在は海面高度で固定）。

### 2. UI/UXの向上

- **弾道データの可視化**: グラフや数値で、最大高度、着弾時間、残存速度などをリアルタイム表示。
- **操作性の改善**: モバイル対応や、より直感的なカメラ操作。

### 3. パフォーマンス最適化

- **Web Workerの活用**: 重い弾道計算をメインスレッドから分離し、描画の滑らかさを維持する。
- **WASM (WebAssembly) 化**: 計算コア部分（PhysicsEngine, ShootingMethodSolver）をRustやC++で書き換え、WASMとして実行することでさらなる高速化を図る。
